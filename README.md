# Student & Book Management API

A simple RESTful CRUD API built with **FastAPI** for managing **students** and **books**. The project demonstrates core FastAPI concepts including routing, request validation with Pydantic, response models, HTTP status codes, exception handling, and database integration.

---

# Project Status

> **Current Implementation Status**
>
> * **Student Module:** Uses an **in-memory Python list** for data storage (non-persistent).
> * **Book Module:** Fully migrated to **SQLite** using **SQLAlchemy ORM**, providing persistent CRUD operations. Book data is now stored in a SQLite database and persists across application restarts.
> * **Authentication:** JWT-based user authentication has been added. Users can sign up and log in, and the `DELETE /books/{book_id}` endpoint is now a protected route requiring a valid access token.

---

# Features

## Student Management

* List all students
* Retrieve a student by ID
* Create a new student
* Update an existing student
* Delete a student

## Book Management

* List all books
* Retrieve a book by ID
* Create a new book
* Update an existing book
* Delete a book
* Persistent storage using **SQLite** and **SQLAlchemy ORM**

## Authentication

* User signup with hashed password storage
* User login returning a JWT access token
* Password hashing using **passlib** (bcrypt)
* JWT token generation and validation using **python-jose**
* Protected endpoint (`DELETE /books/{book_id}`) requiring a valid Bearer token
* Secret key and token settings loaded from environment variables via **python-dotenv**

## Additional Features

* Request validation using **Pydantic**
* Automatic response validation with `response_model`
* Proper HTTP status codes
* Custom error handling using `HTTPException`
* Interactive API documentation (Swagger UI & ReDoc)

# Installation

Clone the repository:

```bash
git clone https://github.com/tcintern-011/Student-Management-API.git
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Set up environment variables:

```bash
cp .env.example .env
```

Then open `.env` and replace the placeholder with a real secret key, generated with:

```bash
openssl rand -hex 32
```

`.env` holds your real `SECRET_KEY` and is never committed to git. `.env.example` is the committed template showing which variables are required.

---

# Running the Application

Start the development server:

```bash
uvicorn main:app --reload
```

The `--reload` flag automatically restarts the server whenever code changes are detected.

# API Endpoints

## Student Endpoints

| Method | Endpoint                 | Description              | Status |
| ------ | ------------------------ | ------------------------ | ------ |
| GET    | `/students`              | Retrieve all students    | 200    |
| GET    | `/students/{student_id}` | Retrieve a student by ID | 200    |
| POST   | `/students`              | Create a new student     | 201    |
| PUT    | `/students/{student_id}` | Update a student         | 200    |
| DELETE | `/students/{student_id}` | Delete a student         | 204    |

---

## Book Endpoints

| Method | Endpoint           | Description             | Status | Auth required                   |
| ------ | ------------------ | ------------------------ | ------ | -------------------------------- |
| GET    | `/books`           | Retrieve all books      | 200    | No                                |
| GET    | `/books/{book_id}` | Retrieve a book by ID   | 200    | No                                |
| POST   | `/books`           | Create a new book       | 201    | No                                |
| PUT    | `/books/{book_id}` | Update an existing book | 200    | No                                |
| DELETE | `/books/{book_id}` | Delete a book           | 204    | **Yes** — requires Bearer token   |

> **Note:** Book endpoints use **SQLite** as the persistent storage backend through **SQLAlchemy ORM**. Changes made through the API are saved to the database and remain available after restarting the application.

> **Note:** All endpoints that retrieve, update, or delete a single resource return **404 Not Found** if the requested ID does not exist.

---

## Auth Endpoints

| Method | Endpoint  | Description                                | Status | Auth required |
| ------ | --------- | -------------------------------------------- | ------ | -------------- |
| POST   | `/signup` | Register a new user (username + password)    | 201    | No             |
| POST   | `/login`  | Log in, returns a JWT access token            | 200    | No             |

> **Note:** `/login` expects `application/x-www-form-urlencoded` data (`username` and `password` fields) rather than JSON, since it uses FastAPI's `OAuth2PasswordRequestForm`. This is also what powers the **Authorize** button in Swagger UI (`/docs`).

> **Note:** `/signup` returns **400 Bad Request** if the chosen username is already taken. `/login` returns **401 Unauthorized** if the username or password is incorrect.

# Validation

FastAPI and Pydantic automatically validate incoming requests.

Examples include:

* Student name must be between **2 and 50 characters**
* Student age must be between **1 and 100**
* Book name must be between **10 and 100 characters**
* Author name must be between **10 and 100 characters**
* ISBN must contain exactly **13 characters**
* Genre must contain between **2 and 20 characters**
* Signup username must be between **3 and 50 characters**
* Signup password must be at least **6 characters**

If validation fails, FastAPI automatically returns a **422 Unprocessable Entity** response with detailed validation errors.
